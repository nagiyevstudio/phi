# Version History

Series: 1.0.x

## Versioning Rules
- Each completed task increments the patch version by 1.
- If a release includes multiple tasks, increase the patch version by the total count of tasks.
- Example: 5 tasks in one release => patch version +5.
- **Single source of truth: `frontend/package.json`** — версия всегда берётся отсюда.
- При каждом бампе версии обязательно обновлять `BUILDS.txt` (через `npm run release:prepare` или вручную по формату ниже).
- Нельзя бампить `package.json` без одновременного обновления `BUILDS.txt`.
