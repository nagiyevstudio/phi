# PerFinance — краткая сводка продукта

## Что это
Веб‑приложение для учета личных финансов: расходы/доходы по категориям, контроль месячного бюджета и быстрый обзор показателей.

## Что реализовано (функционал)
- Регистрация и вход по email/паролю, JWT‑сессия, профиль пользователя (имя, email, роль), выход.
- Операции: создание/редактирование/удаление расходов и доходов с суммой, категорией, датой/временем и заметкой.
- Категории: раздельные списки для расходов и доходов, цвет категории, редактирование, архивирование (soft‑delete).
- Месячный бюджет: план, остаток, перерасход, дневной лимит и траты за сегодня.
- Главный дашборд: быстрый обзор бюджета, дневного лимита и операций за сегодня.
- Операции за месяц: фильтры по типу, категории, диапазону дат и поиску по заметкам/названиям.
- Аналитика по месяцам: итоги (доходы/расходы/итого), расходы по категориям и по дням (графики).
- Экспорт данных: JSON/CSV за месяц или все данные; JSON включает категории, операции и бюджеты.

## Данные и модель
- Пользователи, категории, операции и месячные бюджеты хранятся отдельно для каждого пользователя.
- Суммы хранятся в минорных единицах (копейки/центы) и конвертируются в UI.
- Роль пользователя: `owner` / `editor` / `viewer` (запись разрешена для owner/editor).

## Техническая реализация (кратко)
- Backend: PHP + MySQL/MariaDB, REST API, серверная валидация, PDO prepared statements.
- Frontend: React + TypeScript, React Router, React Query, Tailwind, Recharts.
- Авторизация: JWT, токен хранится в localStorage, авто‑логаут при 401.

## Границы и то, чего нет в коде
- Нет UI для управления ролями/приглашениями пользователей (роль хранится, но управлять ей нельзя).
- Нет импорта данных/банковских интеграций/повторяющихся операций.
- Валюта фиксирована (в интерфейсе отображается AZN), настроек валюты нет.
